# 数据库迁移（一期）

一期只提供最小的建表 SQL：`migrations/init.sql`（PostgreSQL）。

执行方式（示例）：

1. 确保已创建数据库并具备权限
2. 执行 `init.sql`

注意：
- 一期不内置迁移工具，避免引入额外复杂度；由部署脚本或 DBA 执行即可。

## 增量游标（无需单独存储）

本项目不单独存储“游标/水位”：
- 订单增量拉取基于 `dsco_order_sync.dsco_create_time` 的最大值。
- 若 `dsco_order_sync` 为空，则从 `2025-01-01T00:00:00Z`（UTC）开始往后拉取。

如需初始化补数据/回溯：使用 Admin 的“手动拉单入库（指定时间范围 + 入库状态）”功能（范围边界为 `[start, end)`，UTC 秒级时间戳）。
